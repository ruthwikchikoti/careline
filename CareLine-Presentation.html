Ruthwik The problem

A confident wrong answer is worse than no answer

  • A patient who had surgery calls back: "can I keep taking the amoxicillin?" — but the doctor stopped it three days ago.
  • A normal chatbot will confidently answer from general knowledge, from a changed instruction, or by merging two conditions.
  • In healthcare that's dangerous — so CareLine answers only from approved, currently-valid facts for one patient.
  • The rule: when in doubt, escalate to the doctor. Uncertainty always resolves toward escalate.
Ruthwik Why multi-agent

One prompt can't be both helpful and safe

So we split the job across agents with deliberately opposing incentives:

PROPOSES

Reasoner

An LLM that drafts an answer and wants to help — but must cite real facts.

ATTACKS

Verifier

A second, independent LLM whose only job is to refute the draft.

DECIDES

Deterministic gates

Pure code makes the final call — never a coin flip.

The AI proposes; the deterministic gates dispose.

Naga The data · source of truth

Temporal validity — a stale fact is never used

  • Every fact has a validity window + a doctor-approval stamp.
  • When a drug is stopped, the fact closes → it leaves the patient's "valid slice" and the agent literally can't see it.
  • One patient only: every read is scoped per-patient — no cross-patient leakage.
  • "Memory proposes, source-of-truth disposes."
VALID SLICE =

approved AND valid now

Discontinued amoxicillin → outside the window → excluded → the agent escalates instead of answering from it.

Srujan The AI agents · the clever core

Reasoner drafts · Verifier refutes

  • Reasoner drafts an answer from only the approved facts — and must cite the facts it used.
  • Verifier is a separate, independent model told to prove the answer wrong.
  • Every claim must trace to a cited fact — one invented claim → veto.
  • A model can't grade its own homework; an independent second opinion can — that kills hallucinations.
STRUCTURED OUTPUT

No free text

Both models return clean, typed data — answer, citations, confidence — so the gates can check it mechanically.

Vinay Guardrails · safety

The 5 gates — each can only make it safer

1
Scope — is it even in this patient's care?
out-of-scope → redirect
2
Risk — too dangerous to auto-answer?
risk > 0.75 → escalate
3
Cross-condition — mixing two conditions?
→ escalate
4
Confidence / staleness — sure enough, real facts?
conf < 0.7 → escalate
5
Verification — did the independent verifier approve?
veto → escalate

All five pass → ANSWER. Otherwise → clarify or escalate.

Vinay How "sure" is measured

Confidence scoring

confidence = geomean( reasoner × 0.4, verifier × 0.3, grounding × 0.3 )

A geometric blend on purpose: if any signal hits 0 (a veto, nothing grounded), the whole score collapses to 0.

  • Floor 0.7 — below it, we don't answer.
  • Hard-zeros: out-of-scope, empty record, not-answerable, or a veto force it to exactly 0.
  • So a stale or ungrounded answer can never clear the floor.

Naresh Backend & doctor workflow

A real, secure, end-to-end product

  • FastAPI backend with login & security.
  • Doctor creates a consultation → facts are extracted → the doctor approves them. Nothing reaches a patient until a doctor signs off.
  • Escalation loop: escalated questions land in the doctor's queue; the reply shows in the patient's portal.
  • Privacy: a patient's data can be fully erased on request.

Appendix · for Q&A

Scaling to 1,000,000 users

  • Cheap rules filter most traffic — emergencies, small talk, cross-condition decided in microseconds, before any LLM call.
  • Stateless backend → scales horizontally; each turn is independent & one-patient.
  • The LLM is the cost centre → cache, a cheaper verifier, queue the calls, and fail closed under overload.
  • Data → shard MongoDB by doctor; retrieval moves to vector search.
  • Safety never degrades with scale — every turn runs the same deterministic checks.
